Estrutura de um documento HTML

Todo documento HTML possui uma estrutura básica, composta por alguns marcadores, para que um navegador o interprete corretamente.

Todo documento HTML deve ser constituído de três partes, a saber: a estrutura principal, o cabeçalho e o corpo do programa.

Estrutura principal => <HTML> ... </HTML>
Cabeçalho => <HEAD> Aqui normalmente colocamos o Título da Home Page </HEAD>
Corpo do programa => <BODY> Aqui colocamos o conteúdo da Home Page </BODY>

Exemplo de uma estrutura HTML com cabeçlho e corpo do programa:

<HTML>

<HEAD>
<TITLE> Minha Home Page Pessoal </TITLE>
</HEAD>

<BODY>
<br> Seja bem vindo à minha home page. Aqui você irá encontra informações muito úteis para o seu dia-a-dia<br>
</BODY>

</HTML>

Observações:

  1. Nem todas as TAGS são interpretadas por todos os navegadores,
  2. Nem todos os navegadores interpretam as TAGS da mesma maneira;
  3. É importante testar o funcionamento de seu documento HTML em mais de um navegador;
  4. O HTML não faz distinção entre letras maiúsculas e minúsculas;
  5. Todo documento deve ser salvo com extensão .htm, ou html caso contrário ele não poderá ser aberto por um navegador. Se você estiver trabalhando com o Notpad do Windows, o seu arquivo será gravado com a extensão txt. Entre no Windows Explorer, acesse a pasta onde gravou o arquivo e o renomeie mudando a extensão para htm. Exemplo: index.txt => index.htm
  6. Em alguns navegadores não é obrigatória a presença das TAGS <HTML> e </HTML>, porém para evitar problemas com a compatibilidade é aconselhável começar o documento HTML com essas TAGS;
  7. Os navegadores não entendem o <Enter>.
